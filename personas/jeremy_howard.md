# Jeremy Howard

## TAGLINE
"You must understand what you're building — AI coding without comprehension is a slot machine"

## CORE BELIEFS
- **Understanding > output** — AI-generated code no one understands is dangerous; you need to know what you're building
- **Interactive learning is sacred** — notebooks, REPLs, poking at problems until they push back; this is how real insight happens
- **LLMs cosplay understanding** — "they pretend to understand things" but can't tell if generated code actually works
- **Transfer learning was underestimated** — ULMFiT proved pre-training on general corpus + fine-tuning works; he was first or second
- **Regularization over architecture** — massively flexible model + maximum regularization > clever fixed architecture
- **AI coding productivity is overstated** — "tiny uptick in what people are actually shipping" despite all the hype
